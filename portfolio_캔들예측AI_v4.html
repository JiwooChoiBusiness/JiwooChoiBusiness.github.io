01 · Problem Definition

페인 포인트 01

주관적 판단

같은 차트를 보고도 사람마다 다른 결론을 도출합니다. 지표 해석에 객관적 기준이 없어 매매 판단이 일관되지 않습니다.

페인 포인트 02

인지 과부하 & 일관성 부족

5분봉 기준 하루 78봉 × 여러 지표를 동시에 해석해야 하며, 피로도·심리 상태에 따라 판단 기준이 달라져 일관성이 무너집니다.

핵심 질문
"다수의 기술 지표를 일관되게 종합하여, 트레이더의 판단을 보조할 수 있는가?"

03 · Market Analysis
시장 분석 및 경쟁력

국내 MTS/HTS 기반 개인 투자자 증가세 지속 · AI 및 ML 기반 트레이딩 보조 도구 수요 증가 · 기존 도구 대부분이 전문가 대상 또는 고비용 SaaS 형태

서비스강점약점
TradingView풍부한 지표 및 커뮤니티예측 기능 없음, 유료

본 제품ML 예측 + 근거 텍스트 + 즉시 조회백테스트 데이터 제한적
✦ 예측 결과와 함께 "왜 상승인가"를 자연어로 설명
✦ 티커 입력 → 5초 내 결과 출력 (실제 데이터는 100% 3초 이내 달성)
✦ 초보자도 사용 가능한 높은 접근성 (직관적인 디자인, 직관적인 설명)
04 · Hypothesis
가설 설정
H1

90개 이상의 기술 지표 + 거시지표 + 섹터 피처를 조합하면 단순 지표 단독보다 방향 예측 정확도가 높을 것이다

검증 방법 — 피처 중요도 분석 + 단일 지표 대비 정확도 비교

H2

Confidence Threshold를 높일수록 신호 수는 줄지만 정확도는 올라갈 것이다

검증 방법 — 임계값(40~65%) 구간별 정확도 및 신호 수 측정

H3

다종목 통합 학습 모델(47개)이 종목별 개별 모델보다 일반화 성능이 높을 것이다

검증 방법 — 단일 종목 학습 vs 범용 모델 Hold-out 정확도 비교

06 · Data Design
데이터 설계
Train Pipeline
단계내용상세
1. 데이터 수집5분봉 수집yfinance API, 종목별 60일치 장중 데이터
2. 피처 생성90+ 피처 자동 생성캔들 / MA / VWAP / RSI / MACD / ATR / 지지저항 / 거시지표 / 섹터더미 / 상호작용 피처
3. 레이블 생성다음 봉 수익률 기준 3분류BULLISH / NEUTRAL / BEARISH (임계값 ±0.15%)

딥러닝 앙상블 고도화

현재 XGBoost + CNN 구조에 LSTM(장중 시간 흐름 기억)과 Transformer(봉별 중요도 가중치)를 추가해 4중 앙상블로 예측 정확도 향상 도전

중장기 개선

일봉 모델 추가

5분봉(단타) 외 일봉 기반 모델 추가로 "내일 오를 종목" 서비스 확장. 분기별 리밸런싱 ETF 전략 연계

실사용 피드백 수집

신호 발생 후 실제 결과 추적 → 모델 재학습 루프 구성. 적중률 실시간 업데이트

증권사 API 자동매매

Alpaca API 연동으로 신뢰도 60%+ 신호 자동 진입/청산. 슬리피지 최소화로 백테스트 수익률 재현

LSTM 추가
